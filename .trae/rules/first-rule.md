# Project Rules

## Project Rules

- **Name**: 项目名称应与'订单卖车TOB后台Saas服务系统'一致。
- **Goal**: 所有开发活动应围绕'开发一个订单卖车的TOB后台Saas服务系统'这一目标。
- **Background Considerations**: 开发过程中需充分考虑'移动互联网普及，传统销售模式无法满足需求'的项目背景。

## Tech Stack Rules

### Frontend

- **Framework**: 前端必须使用React框架进行开发。
- **Version**: 推荐使用最新稳定版React，并遵循其最佳实践。

### Backend

- **Framework**: 后端必须使用Node.js进行开发。
- **Version**: 推荐使用LTS版Node.js，并遵循其异步编程最佳实践。

## Functional Module Rules

- **General**: 所有功能模块的开发都应遵循模块化原则，确保高内聚、低耦合。

### 用户登录模块

- 用户注册功能需包含必要的字段验证和安全性考虑（如密码加密）。
- 用户登录功能需支持多种认证方式（如用户名/密码、手机号/验证码），并处理会话管理。
- 用户权限管理需实现基于角色的访问控制（RBAC），明确区分不同用户角色的操作权限。

### 车辆管理模块

- 车辆信息录入界面需友好，支持批量导入，并对数据进行严格校验。
- 车辆信息查询功能需支持多条件筛选、分页、排序，并提供高效的查询响应。
- 车辆状态管理需能清晰反映车辆的销售、库存、维修等状态流转，并支持状态变更记录。

### 订单管理模块

- 订单创建流程需完整，包括商品选择、价格计算、优惠应用、用户确认等环节。
- 订单查询功能需支持按用户、车辆、时间、状态等多维度查询，并提供订单详情。
- 订单状态管理需涵盖从创建到完成/取消的所有状态，并支持状态流转的自动化或手动操作。

### 基础数据模块

- 所有基础数据（品牌、型号、颜色、状态等）应通过后台统一管理界面进行维护。
- 基础数据应支持增删改查操作，并提供数据导入导出功能。
- 基础数据应具备版本管理或历史记录功能，以便追踪变更。

### 支付模块

- 支付订单功能需集成主流支付渠道（如支付宝、微信支付），确保支付安全性和稳定性。
- 查询支付订单状态功能需能实时同步支付渠道的支付结果，并更新本地订单状态。

## Backend Design Rules

### Database Design

- **General**: 数据库设计应满足三范式要求，确保数据一致性和完整性。
- **Naming Convention**: 数据库表名和字段名应采用小写蛇形命名法（snake_case）。
- **Tables**:
  - 用户表：应包含用户基本信息、认证信息等。
  - 车辆表：应包含车辆详细参数、库存信息等。
  - 订单表：应包含订单基本信息、关联用户和车辆信息、支付状态等。
  - 支付表：应记录支付流水、支付状态、交易ID等。
  - 基础数据表：应为品牌、型号、颜色、状态等基础信息提供独立的管理表。
- **Technology**: 数据库必须使用MySQL。

### API Design

- **Style**: 所有后端API必须遵循RESTful风格。
- **Endpoints**: API端点应使用名词复数形式表示资源集合，例如 /users, /vehicles。
- **Methods**: 正确使用HTTP方法（GET用于查询，POST用于创建，PUT用于更新，DELETE用于删除）。
- **Response Format**: API响应应统一使用JSON格式，并包含状态码、消息和数据。

### Architecture Design

- **Pattern**: 后端架构必须采用MVC（Model-View-Controller）模式，明确划分职责。
- **Type**: 系统架构为单体架构，但需考虑未来微服务化的扩展性。

## Frontend Design Rules

- **Framework**: 前端必须采用React框架进行开发，遵循组件化、声明式UI原则。
- **Routing**: 前端路由必须采用react-router-dom，实现SPA（单页应用）的路由管理。
- **State Management**: 前端状态管理必须采用Redux，遵循单向数据流原则，使用Redux Toolkit简化开发。
- **UI Component Library**: 前端UI组件库必须采用Ant Design，确保界面风格统一和开发效率。
- **Deployment**: 前端部署必须采用Nginx，负责静态资源服务、路由转发和负载均衡。
